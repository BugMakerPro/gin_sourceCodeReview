#### 1. why start?
> 刚从Java转go的时候,在公司主要做基础设施方面的开发,当时用go做的第一个项目就是一个web框架.  
> 是基于```httprouter```做的二次开发,支持常见的web框架的要求,比如middleware等，  
> 同时还加入了日志和存储等方面的实现,总体来看是一个重复造轮子的项目。  
> 当时没有来得及去了解go的其他web框架的实现,所以对比很多JD,
> 总觉得自己在go web框架方面还有技术盲点,不免有些遗憾,所以决定把gin和beego等的源码阅读补上.
